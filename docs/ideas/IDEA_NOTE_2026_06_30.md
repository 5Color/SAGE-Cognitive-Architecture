# SAGE Idea Note - 2026-06-30

## Source

센터에 있던 친구와 피드백을 주고받으며 함께 정리한 아이디어 노트.

## Core Direction

SAGE의 구조는 자연 생태계형이지만, 기능적 특징은 인간의 인지 시스템을 부분적으로 모방하는 방향으로 확장할 수 있다.

SAGE는 단순히 여러 Organ이 있는 시스템이 아니라, 상태, 기억, 반성, 휴식, 자기검증, 지속학습, 통찰을 통해 점진적으로 성장하는 인공 인지 생태계를 목표로 한다.

---

## 1. Brain Plasticity and Memory Degradation

SAGE는 뇌 가소성에서 영감을 얻어, 경험에 따라 Organ의 중요도나 선택 경향이 변화하는 구조를 가질 수 있다.

또한 인간의 망각이나 인지 저하 현상처럼, 오래되었거나 검증되지 않았거나 낮은 가치의 memory가 약해지는 구조도 고려할 수 있다.

핵심 연구 질문:

```text
SAGE의 memory는 시간이 지나면서 어떻게 약해지는가?
사용하지 않는 knowledge는 어떻게 낮은 우선순위로 내려가는가?
반복적으로 유용했던 knowledge는 어떻게 강화되는가?
잘못된 memory가 누적되면 시스템 성능이 저하되는가?
```

---

## 2. Dialectic Reflection and Personality Formation

SAGE는 하나의 답을 바로 내는 것이 아니라, 두 개 이상의 내부 관점이 논리적으로 토론하는 구조를 가질 수 있다.

예를 들어 하나의 Organ은 주장을 만들고, 다른 Organ은 반론을 만들고, Judge Organ은 양쪽을 비교한다.

```text
Proposer Organ
- 주장 생성

Opponent Organ
- 반론 생성

Judge Organ
- 논리적 비교

Memory Organ
- 토론 결과 저장
```

이런 반복된 판단 패턴은 장기적으로 SAGE의 성향이나 personality state처럼 작동할 수 있다.

---

## 3. Self-Model and Self-Questioning

SAGE가 실제 자아를 가진다고 주장하지는 않는다.

그러나 SAGE가 자기 상태, 한계, 목표, 판단 근거를 모델링하는 구조는 연구할 수 있다.

핵심 질문:

```text
SAGE는 자기 상태를 설명할 수 있는가?
SAGE는 자기 판단의 약점을 찾을 수 있는가?
SAGE는 스스로 질문을 생성할 수 있는가?
SAGE는 자신의 이전 판단을 반성할 수 있는가?
```

이 방향은 self-awareness가 아니라 self-modeling과 meta-cognition으로 다룬다.

---

## 4. Language Processing and State-Based Understanding

SAGE는 언어를 단순한 token 나열로만 처리하지 않고, 상태 변화와 의미 구조로 변환하는 방식을 고려한다.

예를 들어 문장을 다음과 같이 해석할 수 있다.

```text
문장:
나는 AGI를 만들고 싶다.

상태 해석:
speaker = 나
goal = AGI 제작
desire = 높음
action_need = 연구와 개발
```

이 방식은 인간이 문장을 이해할 때 단어만 보는 것이 아니라, 상황, 의도, 감정, 목표를 함께 추상화하는 방식과 닮아 있다.

---

## 5. Recursive Learning, Continual Learning, and Insight

SAGE는 이미 알고 있는 지식을 다시 연결하고, 새로운 가설을 생성하는 구조를 가질 수 있다.

다만 자기 생성 데이터를 무제한으로 다시 학습하는 방식은 위험할 수 있다.

따라서 SAGE의 지속학습은 다음 단계를 거쳐야 한다.

```text
observation
reflection
memory proposal
human approval
reuse
contradiction check
experiment proposal
```

SAGE의 insight는 즉시 truth로 저장되지 않고, hypothesis로 저장되어야 한다.

---

## 6. Rest Mode, Sleep Mode, and Sudden Ideas

SAGE는 항상 실행 상태일 필요가 없다.

인간이 쉬거나 잠자는 동안 기억을 정리하고 새로운 연결을 만드는 것처럼, SAGE도 rest mode를 가질 수 있다.

rest mode에서는 외부 행동을 하지 않고 내부적으로 다음 작업을 수행한다.

```text
memory review
weak signal connection
old result summary
new hypothesis proposal
contradiction scan
```

휴식 중 갑작스러운 아이디어는 무작위 출력이 아니라, 기존 memory와 reflection이 새롭게 연결된 hypothesis로 다룬다.

---

## 7. Arousal and Hormone-Like State Variables

SAGE는 인간의 호르몬을 그대로 모방할 필요는 없지만, 내부 조절 변수는 가질 수 있다.

예시:

```text
curiosity
caution
fatigue
confidence
urgency
stability
exploration
```

이 변수들은 Router나 Aggregator의 선택에 영향을 줄 수 있다.

예를 들어 curiosity가 높으면 탐색적인 Organ이 더 선택되고, caution이 높으면 Critic Organ이 더 강하게 작동할 수 있다.

---

## 8. K-AI and Korean-Optimized Token Processing

SAGE의 중요한 장기 목표 중 하나는 K-AI 발전에 기여하는 것이다.

한국어와 한글은 AI에 불리한 언어가 아니라, 한국어 구조에 맞는 tokenizer와 memory 구조가 충분히 활용되지 않았을 수 있다.

한글은 자모, 음절, 어절 구조를 가지고 있고, 한국어는 조사, 어미, 형태소 결합을 통해 많은 의미와 문법 정보를 압축한다.

따라서 SAGE는 장기적으로 다음을 실험할 수 있다.

```text
jamo-level token
syllable-level token
morpheme-aware token
eojeol-level token
meaning chunk token
state-based semantic token
```

목표는 단순히 token 수를 줄이는 것이 아니라, 적은 token으로 의미, 문맥, 문법 정보, reasoning 단서를 안정적으로 보존하는 것이다.

## 10. Security Learning Direction

SAGE는 해킹을 공격 목적이 아니라 윤리적 보안 학습과 방어 중심으로 다룬다.

가능한 방향:

```text
Linux command learning
network basics
web security concepts
CTF practice
log analysis
defense explanation
secure coding
```

금지 방향:

```text
unauthorized hacking
real target attack
malware creation
credential theft
stealth or evasion
```

## 11. Philosophy and Mathematical Interpretation

SAGE는 기존 철학자와 사상가들의 문장을 단순히 인용하는 것이 아니라, 그 주장을 의미 분석하고 구조화하여 수학적 또는 논리적 표현으로 변환하는 방향을 고려할 수 있다.

예시:

```text
철학적 주장
↓
의미 단위 분해
↓
전제 / 결론 / 가치 판단 분리
↓
논리 구조 표현
↓
SAGE reflection policy나 decision rule에 적용
```

이 방향은 철학을 감성적 문장으로만 보는 것이 아니라, reasoning policy로 변환하는 시도이다.
